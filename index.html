What is MongoDB?

MongoDB introduces the most popular document-oriented database management system at the moment.

>

Documents instead of lines

If relational databases store rows, then MongoDB stores documents. Unlike lines, documents can store structurally complex information. The document can be represented as a key and value store.

The key represents a simple label with which a particular piece of data is associated.

A specific value is mapped to each key. If a value is not associated with any key, then this key is simply omitted in the document and is not used.

Collections

If there are tables in the traditional SQL world, then there are collections in the MongoDB world

Replication

The storage system in MongoDB represents a set of replicas. There is a primary node in this set and there may also be a secondary node set.

If the main node for some reason fails, then one of the secondary nodes becomes the main one.

Data format in MongoDB

BSON allows you to work with data faster: faster search and processing

Cross-platform

MongoDB is written in C++, so it is easy to port to a wide variety of platforms.

GridFS

MongoDB offers a special GridFS technology that allows you to store data in size more than 16 MB
